# Checkpoint 템플릿

wiki-writer 에이전트가 Checkpoint 페이지를 작성할 때 이 템플릿을 따른다.

## 페이즈 정의

| Phase | 기간 | 핵심 산출물 |
|-------|------|------------|
| 1: 데이터 준비 | 4/8 | DONGNE_MASTER, 4축 피처 확정, EDA |
| 2: Cortex AI 파이프라인 | 4/9 | MBTI 분류, Semantic Model, Agent |
| 3: Streamlit UI | 4/10 | 3탭 앱 동작 |
| 4: 테스트 + 제출 | 4/11~12 | E2E, 데모, PPT, ZIP |

## 페이지 템플릿

```markdown
# Checkpoint: Phase {N} — {페이즈명}

> 기간: {시작}~{종료} | 크레딧 사용: ~${사용} (누적 ${누적}/$40)

## 목표 vs 실제
| 목표 | 달성 | 비고 |
|------|------|------|
| {목표1} | 완료/부분/미착수 | {설명} |

## 핵심 산출물
1. **{산출물}**: {설명 + 파일 경로}

## 데이터 분석 결과 (해당 시)
{EDA 결과 요약, 피처 매핑 결과 등}

## 기술 의사결정
- {결정}: {이유}

## 리스크 업데이트
| 리스크 | 이전 상태 | 현재 상태 | 대응 |

## 다음 Phase 계획
- {작업 목록}
```

## Decision 템플릿

```markdown
# Decision: {제목}

> 날짜: {YYYY-MM-DD} | 관련 Phase: {N}

## 맥락
{왜 이 결정이 필요했는가}

## 선택지
1. **{옵션 A}**: {장단점}
2. **{옵션 B}**: {장단점}

## 결정
{선택한 옵션과 이유}

## 결과
{결정 후 어떻게 되었는가 — 나중에 업데이트}
```
