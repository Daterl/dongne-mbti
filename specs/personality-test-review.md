## 누락 항목

### Critical (0개)

없음.

### Important (6개)

1. **점수 스케일 불일치**: 사용자 점수(~[-0.6, +0.6])와 동네 z-score(~[-2, +2])의 범위가 다릅니다. `match_neighborhoods()`가 유클리드 거리를 쓰면, 모든 사용자가 중심부 근처 동네에만 매칭될 수 있어요.
   - 왜 필요한가: 극단적 성향의 사용자도 극단적 동네에 매칭되어야 결과가 의미 있습니다. 스케일링 계수를 적용하거나, 유클리드 거리 대신 코사인 유사도를 검토해야 합니다.

2. **"뒤로 가기" / "다시 하기" 미정의**: 질문 중 이전 질문으로 돌아가는 방법, 결과 화면에서 다시 시작하는 방법이 스펙에 없습니다.
   - 왜 필요한가: 데모 중 심사위원이 "아 잘못 골랐는데" 하면 앱을 새로고침해야 합니다. 최소한 결과 화면의 "다시 하기" 버튼은 필수입니다.

3. **Cortex Search 역할 모호**: 파이프라인에 "프로필 텍스트 기반 동네 매칭 보조"로 기재되어 있지만, 실제 매칭은 `match_neighborhoods()`의 순수 Python 유클리드 거리입니다. Cortex Search가 구체적으로 언제, 어떻게 호출되는지 불명확합니다.
   - 왜 필요한가: 심사에서 "6개 Cortex 기능을 모두 활용했는가"를 봅니다. 호출이 없으면 빈 파이프라인이 됩니다. Cortex Search를 추천 이유 생성의 context 보강에 활용하거나, 매칭 후 프로필 텍스트 조회에 사용하는 것을 명시해야 합니다.

4. **결과 페이지 로딩 시간**: AI_COMPLETE를 5-6회 순차 호출하면 최소 10-30초가 걸립니다. 이 동안 사용자가 보는 화면(로딩 상태)이 정의되지 않았습니다.
   - 왜 필요한가: 데모 중 30초 빈 화면은 치명적입니다. Phase A/B/C를 점진적으로 렌더링하거나, 먼저 정적 결과를 보여주고 AI 텍스트를 나중에 채우는 전략이 필요합니다.

5. **테스트 가능한 성공 기준 부재**: "심사 기준 대응"은 있지만, "8개 질문 완료 → 3개 동네 표시 → 각 동네에 AI 텍스트 → 시세 차트"처럼 검증 가능한 체크리스트가 없습니다.
   - 왜 필요한가: 구현 완료 시점을 판단할 기준이 필요합니다.

6. **기존 코드 마이그레이션 누락**: `st.experimental_rerun()` → `st.rerun()` 교체가 SiS 주의사항에 언급되어 있지만, 기존 app.py의 6곳 교체가 파일 변경 사항에 포함되지 않았습니다.
   - 왜 필요한가: 교체하지 않으면 기존 탭들이 깨질 수 있습니다.

### Nice-to-have (3개)

1. **범위 밖(Out-of-scope) 명시 부재**: 이 탭에서 안 하는 것이 명시되어 있지 않습니다 (예: 결과 저장, 공유 링크, 재방문 시 이전 결과 표시 등).

2. **동점 처리**: 유클리드 거리가 동일한 동네가 여러 개일 때의 정렬 기준이 없습니다.

3. **프로필 데이터 없는 경우**: `load_profiles()`가 빈 DataFrame을 반환할 때의 빈 상태(empty state) UI가 정의되지 않았습니다.

## 발견된 엣지 케이스

- **극단적 답변 패턴**: 8개 질문 모두 같은 위치(예: 모두 A)를 선택하면 특정 축에 과도하게 편향된 점수가 나옵니다. 매칭 결과가 극소수 동네에 집중될 수 있습니다.
- **탭 전환 복귀**: 퀴즈 진행 중 다른 탭으로 갔다 오면 `quiz_step`이 유지되므로 기능적 문제는 없으나, CSS 애니메이션이 다시 재생됩니다 (의도와 무관하게).
- **AI 호출 전체 실패**: 3단계 폴백 체인이 정의되어 있으므로 앱이 깨지지는 않으나, 정적 텍스트만 나오면 "AI 전문성" 심사 점수에 영향이 있습니다.

## 권장 사항

1. **점수 스케일링 추가** — `compute_user_scores()` 결과에 ×3 스케일링 팩터를 적용하여 동네 z-score 범위(-2~+2)에 맞추기. 또는 코사인 유사도 사용.
2. **"다시 하기" 버튼 추가** — 결과 화면 하단에 `reset_quiz()` 호출 버튼 배치.
3. **Cortex Search 활용처 명확화** — TOP 3 매칭 후, 각 동네의 PROFILE_TEXT를 Cortex Search로 조회하여 AI_COMPLETE의 context로 주입. 이렇게 하면 자연스러운 파이프라인이 됩니다.
4. **점진적 렌더링** — Phase A(DNA)를 먼저 표시 후 spinner로 Phase B(TOP 3)를 로딩. Phase B 각 동네도 하나씩 표시하여 체감 대기 시간 단축.
5. **성공 기준 추가** — "인트로 → 8문제 → 분석 → MBTI 카드 + DNA 텍스트 + TOP 3 카드 + 시세 차트 → 모두 렌더링"을 DoD로.
6. **기존 rerun 교체** — app.py 6곳의 `st.experimental_rerun()` → `st.rerun()` 교체를 파일 변경 사항에 명시.
